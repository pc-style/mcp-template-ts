# MCP Autonomous Development Team

A comprehensive Model Context Protocol (MCP) server that enables clients (Cursor, Claude Code, etc.) to delegate tasks to specialized subagents forming an autonomous development team. The system takes a Product Requirements Document (PRD) and produces a working prototype through coordinated agent collaboration.

## 🚀 Features

### Core Capabilities
- **Complete Project Orchestration**: From PRD to working prototype
- **Multi-Agent Architecture**: Specialized agents for each development role
- **Comprehensive Toolset**: 9 MCP tools covering the entire development lifecycle
- **Multiple LLM Providers**: Support for OpenAI, Anthropic, and local models
- **Production-Ready Output**: Clean, tested, and deployable code
- **Full Documentation**: Auto-generated project documentation

### Agent Team Structure
- **Product Manager**: Analyzes PRDs, creates user stories, prioritizes requirements
- **Tech Lead**: Designs architecture, defines tech stack, creates API specs
- **Developer**: Implements features, writes clean code, follows best practices
- **QA Engineer**: Creates test plans, writes comprehensive tests
- **DevOps Engineer**: Sets up deployment, CI/CD, monitoring, security
- **Orchestrator**: Coordinates all agents and manages workflow

## 🛠️ Installation

### Prerequisites
- Node.js 18+ 
- npm or pnpm
- Git (optional, for version control)

### Quick Start
```bash
# Clone the repository
git clone <repository-url>
cd mcp-autonomous-dev-team

# Install dependencies
npm install

# Build the project
npm run build

# Set up environment variables
cp .env.example .env
# Edit .env with your API keys and configuration
```

### Environment Configuration
Create a `.env` file with the following variables:

```env

ANTHROPIC_API_KEY=your_anthropic_api_key
ANTHROPIC_MODEL=claude-3-sonnet-20240229

# Local LLM (optional)
LOCAL_LLM_ENDPOINT=http://localhost:11434

TL_TEMPERATURE=0.5

QA_TEMPERATURE=0.4
```

## 📖 Usage

### MCP Tools

The server provides 9 comprehensive tools:

#### 1. `orchestrate_project` - Complete Project Orchestration
Takes a PRD and orchestrates the entire development process from requirements to deployment.

```json
{
  "prd_content": "Build a task management application...",
  "project_name": "TaskMaster Pro",
  "project_description": "A comprehensive task management solution"
}
```

#### 2. `analyze_requirements` - Requirements Analysis
Analyzes PRD content and extracts structured requirements.

```json
{
  "prd_content": "The system should allow users to...",
  "project_name": "My Project",
  "project_description": "Project description"
}
```

#### 3. `create_architecture` - Architecture Design
Creates system architecture based on requirements and user stories.

```json
{
  "requirements": [...],
  "user_stories": [...],
  "project_constraints": {...}
}
```

#### 4. `implement_feature` - Feature Implementation
Implements specific features based on user stories and technical specifications.

```json
{
  "feature_id": "user_auth",
  "user_stories": [...],
  "tech_stack": {...},
  "api_specs": [...],
  "database_schema": {...},
  "project_path": "./project"
}
```

#### 5. `create_tests` - Test Creation
Creates comprehensive test suites for features.

```json
{
  "features": [...],
  "tech_stack": {...},
  "test_types": ["unit", "integration", "e2e"],
  "project_path": "./project"
}
```

#### 6. `setup_deployment` - Deployment Setup
Sets up deployment configuration and infrastructure.

```json
{
  "tech_stack": {...},
  "deployment_type": "docker",
  "project_path": "./project",
  "environment": "development"
}
```

#### 7. `review_code` - Code Review
Performs comprehensive code review and provides recommendations.

```json
{
  "code_content": "function processData() {...}",
  "language": "typescript",
  "context": "Data processing module",
  "standards": ["eslint", "security"]
}
```

#### 8. `update_documentation` - Documentation Update
Updates and generates project documentation.

```json
{
  "project_context": {...},
  "documentation_type": "readme",
  "project_path": "./project"
}
```

#### 9. `get_project_status` - Project Status
Gets current project status and progress.

```json
{}
```

### MCP Resources

#### `project://{project_id}` - Project Context
Returns the current project context and state.

#### `docs://{project_id}/{doc_type}` - Project Documentation
Returns project documentation by type (readme, api, setup, etc.).

### MCP Prompts

#### `review-code` - Code Review Prompt
Provides structured prompts for code review tasks.

#### `review-architecture` - Architecture Review Prompt
Provides structured prompts for architecture review.

#### `validate-requirements` - Requirements Validation Prompt
Provides structured prompts for requirements validation.

### Development Workflow
1. **Requirements Analysis** → Product Manager analyzes PRD
2. **Architecture Design** → Tech Lead creates system design
3. **Development** → Developer implements features
4. **Testing** → QA Engineer creates and runs tests
5. **Deployment** → DevOps Engineer sets up infrastructure
6. **Completion** → Orchestrator validates and delivers

## 🛡️ Security

### Sandbox Mode
The system runs in sandbox mode by default, restricting file system access and command execution.

### Allowed Commands
Only pre-approved commands can be executed:
- Package managers: npm, yarn, pnpm, pip, poetry
- Build tools: go, cargo
- Version control: git
- Containerization: docker, docker-compose
- Orchestration: kubectl

### Blocked Commands
Dangerous commands are blocked:
- File deletion: rm, del
- System commands: format, dd, mkfs, fdisk
